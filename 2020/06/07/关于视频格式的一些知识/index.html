封装格式 and 媒体格式/编码方式

一个视频，包括画面、音频、章节，有时候还有字幕 这时就需要一个“容器”（封装格式）将它们打包在一起，每一件物品如视频、音频，称之为轨道(track) 每个轨道，都有自己的格式（媒体格式/编码方式） 就比如平常所说的视频H.264 音频是AAC

常见封装格式

1.MP4

MP4即MPEG-4，指MPEG标准第四部分

——MP3其实是MPEG-1 Audio Layer 3

H.264为MP4的一种高清编码 又名MPEG P10 AVC 当然其他容器也可以用

X264为该编码的FourCC（还有AVC1/H264 不带点） 理解成一种算法

H.264的Profile和Level

Profile: Baseline Profile 底线Profile

​ Main Profile 主要Profile

​ High Profile 高端Profile 越往后算法越复杂 压缩比更高

Level:码率信息（这个在Pr好像有） Profile和Level越高对硬件要求越高

HEVC/H.265（比起H.264更为强悍的编码方式）

Hi10p=x264 10bit
Ma10p=x265 10bit

2.MKV

优势：可以封入多达16条音轨 还可以封装字幕

3.FLV 全称Flash Video 网络传输最广的格式

优势：体积很小 同样也可以使用H.264 使用H.264后的名字叫F4V
